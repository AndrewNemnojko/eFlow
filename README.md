# eFlow 
<h1>Automated Flower Bouquet Evaluation System</h1>

<hr>

<h2>Overview</h2>
<p>This project aims to address and rethink the evaluation process of complex products (specifically flower bouquets) on a WordPress site. The current issues faced include:</p>
<ul>
    <li>Each bouquet is available in multiple variations (sizes).</li>
    <li>Information about the composition of each variation (types and quantities of flowers) is missing and calculated manually by staff.</li>
    <li>Inventory control and cost calculation of variations are performed manually.</li>
</ul>
<p>Given the large number of bouquets (thousands) and their multiple variations (3-5 each), updating and changing data efficiently is impossible with the current system.</p>

<hr>

<h2>Features</h2>
<ul>
    <li><strong>Detailed Composition Information</strong>: Each bouquet variation will include detailed information about its composition (types and quantities of flowers).</li>
    <li><strong>Flower Data Management</strong>: Each flower will have data regarding its availability, cost, and seasonality.</li>
    <li><strong>Automated Revaluation</strong>: Automatic revaluation of all bouquets and their variations when flower data is updated.</li>
    <li><strong>Efficient Processing</strong>: The process, which previously required many person-hours, will be executed in minutes via server-side processing.</li>
</ul>

<hr>

<h2>Benefits</h2>
<ul>
    <li><strong>Efficiency Improvement</strong>: Significant improvement in work efficiency by automating manual processes.</li>
    <li><strong>Inventory Management</strong>: Simplified inventory management with real-time data on flower availability and costs.</li>
    <li><strong>Cost Management</strong>: Accurate and quick cost calculation for each bouquet variation.</li>
</ul>
